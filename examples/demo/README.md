# SkeinRank demo

This folder contains a tiny self-contained demo corpus for SkeinRank.

Files:

- `demo_documents.jsonl` — 10 short technical documents
- `demo_queries.jsonl` — 10 short queries with `relevant` document ids
- `demo_enriched_documents.jsonl` — generated by the enrichment command
- `demo_eval_results.json` — generated by the evaluation command

## Enrich the demo corpus

Run from `packages/skeinrank-core`:

```bash
poetry run skeinrank-enrich-jsonl \
  ../../examples/demo/demo_documents.jsonl \
  ../../examples/demo/demo_enriched_documents.jsonl
```

## Evaluate baseline vs normalized attributes

Run from `packages/skeinrank-core`:

```bash
poetry run skeinrank-eval-demo \
  ../../examples/demo/demo_queries.jsonl \
  ../../examples/demo/demo_enriched_documents.jsonl \
  --out ../../examples/demo/demo_eval_results.json
```

The enriched output includes the profile snapshot metadata and the alias matcher backend so it is clear which terminology version produced the attributes.

The output report is intentionally small and human-readable. It is meant to show:

- what attributes were extracted from queries and documents
- how alias canonicalization changes matching
- how a normalized attribute-aware score compares with a plain lexical baseline

The current demo query set intentionally includes a few alias-heavy queries such as `pg slowdown`, `k8s metrics collectors`, and `asp.net null ref in prod` so the normalized path is easier to inspect. On the bundled toy corpus, the expected summary is:

- baseline top-1 accuracy: `0.8`
- normalized top-1 accuracy: `1.0`
- baseline MRR: `0.8833`
- normalized MRR: `1.0`
