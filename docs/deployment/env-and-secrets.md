# Environment validation and secrets

The file-based environment preflight catches unsafe `.env` values for the production-oriented Docker Compose profile before containers start.

The validator is side-effect free: it reads a `.env` file, checks required settings, checks common placeholder secrets, and prints a JSON report. It does not connect to PostgreSQL, RabbitMQ, Elasticsearch/OpenSearch, or the Governance API.

## Basic flow

From the repository root:

```bash
cp .env.production.example .env
# edit .env and replace CHANGE_ME values
make prod-env-check
make prod-config
make prod-up
make prod-smoke
```

Strict mode fails on warnings as well as errors:

```bash
make prod-env-check-strict
```

Direct package command:

```bash
cd packages/skeinrank-governance-api
poetry run python -m skeinrank_governance_api.env_validation validate --file ../../.env
poetry run python -m skeinrank_governance_api.env_validation validate --file ../../.env --strict
```

Installed script equivalent:

```bash
poetry run skeinrank-governance-env validate --file ../../.env
```

## What is checked

The `production-compose` profile checks:

```text
required values
production/auth guardrails
wildcard CORS
placeholder example domains
placeholder or weak secrets
boolean value syntax
absolute UI API URL
optional Elasticsearch/OpenSearch URL and credentials consistency
JSON log format recommendation
```

Required production Compose values include:

```text
POSTGRES_DB
POSTGRES_USER
POSTGRES_PASSWORD
RABBITMQ_DEFAULT_USER
RABBITMQ_DEFAULT_PASS
SKEINRANK_ENV
SKEINRANK_GOVERNANCE_API_ENV
SKEINRANK_GOVERNANCE_API_AUTH_ENABLED
SKEINRANK_GOVERNANCE_API_PRODUCTION_SECURITY_ENABLED
SKEINRANK_GOVERNANCE_API_ADMIN_PASSWORD
SKEINRANK_GOVERNANCE_API_CORS_ORIGINS
VITE_SKEINRANK_GOVERNANCE_API_URL
GRAFANA_ADMIN_PASSWORD
```

`GRAFANA_ADMIN_PASSWORD` is required because Docker Compose interpolates the optional Grafana service even when the `observability` profile is not started.

## Image version policy

Environment validation does not pull Docker images, but the production Compose profile pins datastore image versions explicitly:

```text
postgres:16.4-alpine
rabbitmq:3.13.7-management
```

Keep these versions stable for a pilot/release cycle and bump them deliberately in a separate dependency update.

## Secrets policy

Do not commit `.env`.

Tracked files:

```text
.env.example
.env.production.example
```

Ignored files:

```text
.env
.env.*
```

except the two example templates above.

Use unique secrets for:

```text
POSTGRES_PASSWORD
RABBITMQ_DEFAULT_PASS
SKEINRANK_GOVERNANCE_API_ADMIN_PASSWORD
GRAFANA_ADMIN_PASSWORD
SKEINRANK_GOVERNANCE_API_ELASTICSEARCH_PASSWORD
SKEINRANK_GOVERNANCE_API_ELASTICSEARCH_API_KEY
```

For pilots, use at least 16 characters. For production, prefer 32+ random characters generated by a password manager or secrets manager.

## Elasticsearch/OpenSearch

Elasticsearch/OpenSearch is optional for first bootstrap:

```text
SKEINRANK_GOVERNANCE_API_ELASTICSEARCH_URL=
```

This lets the API, DB, schema health, UI, metrics, troubleshooting report, and backup commands start before search is configured.

When a real endpoint is available, set:

```text
SKEINRANK_GOVERNANCE_API_ELASTICSEARCH_URL=https://your-search.example.com:9200
SKEINRANK_GOVERNANCE_API_ELASTICSEARCH_USERNAME=...
SKEINRANK_GOVERNANCE_API_ELASTICSEARCH_PASSWORD=...
# or
SKEINRANK_GOVERNANCE_API_ELASTICSEARCH_API_KEY=...
```

Do not leave `https://elasticsearch.example.com:9200` in `.env`. Placeholder URLs keep `/readyz` degraded and make strict smoke checks fail.

## Report shape

The validator prints JSON:

```json
{
  "format_version": "skeinrank.env_validation.v1",
  "status": "warning",
  "strict": false,
  "profile": "production-compose",
  "errors": [],
  "warnings": [],
  "summary": {
    "errors": 0,
    "warnings": 0,
    "elasticsearch_configured": false
  }
}
```

Status meanings:

```text
ok       no errors or warnings
warning  no errors, but some values should be reviewed
failed   errors are present, or strict mode found warnings
```
